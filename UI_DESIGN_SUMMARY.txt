================================================================================
USAGE INSTRUCTIONS FOR AGENTS/DEVELOPERS
================================================================================

1. DashLayout.fxml is the MAIN container - loaded first
   - Contains header and user info
   - Has fx:id="mainContent" (AnchorPane) for content injection

2. To Load DashBoard.fxml into mainContent:
   - Use FXMLLoader to load DashBoard.fxml
   - Set it as the child of mainContent AnchorPane
   - Example code in DashLayoutController

3. Both files use the SAME stylesheet: DashBoard.css
   - Provides consistent styling across the application
   - All style classes are properly defined

4. Style Classes Available:
   - .sidebar-button (Navigation buttons)
   - .btn-primary, .btn-secondary, .btn-danger, .btn-success
   - .text-field, .text-area (Input controls)
   - .table-view, .combo-box
   - .label-header, .label-dark

================================================================================
NOTES FOR FUTURE DEVELOPMENT
================================================================================

1. The mainContent AnchorPane in DashLayout.fxml has:
   - fx:id="mainContent" - for programmatic access
   - styleClass="center-content" - for CSS styling
   - VBox.vgrow="ALWAYS" - expands to fill available space

2. DashBoard.fxml is a self-contained module with:
   - Its own controller: DashBoardController
   - BorderPane layout (Sidebar + Center)
   - Can be easily replaced or hidden

3. Header styling in DashLayout.fxml uses:
   - Gradient background (dark blue to slate)
   - User labels (fx:id="userNameLabel", "userRoleLabel")
   - Logout button (fx:id="logoutButton")

4. CSS classes are organized by component type for easy maintenance
   - Search by ".sidebar-", ".btn-", ".table-", etc.
